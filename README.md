# 预设编辑器 (Preset Editor) · SillyTavern 扩展

一个在 SillyTavern 内部直观编辑 / 新建「聊天补全(Chat Completion)预设」提示词条目的工具。

## 安装

把整个 `preset-editor` 文件夹放到：

```
SillyTavern/public/scripts/extensions/third-party/preset-editor/
```

然后刷新浏览器。在右上角「魔棒(扩展)菜单」里会出现 **预设编辑器** 入口（若菜单里没有，会在右下角出现一个浮动圆钮作为回退）。

> 也可以用 SillyTavern 设置里的「Install extension」从 Git 仓库安装，效果相同。

## 使用前提

- 当前 API 需切换到 **Chat Completion（聊天补全）** 模式，且已选中一个预设。
- 若提示「未找到预设数据」，多半是当前为 Text Completion 模式。

## 三个标签页

1. **编辑器** — 列出当前预设全部条目：拖动手柄排序、开关启用、展开编辑（名称 / 角色 / 位置 / 深度 / 内容 / 禁止覆盖）、复制、删除、新建、重新添加已移除的条目、搜索过滤。
2. **教程** — 逐项解释每个字段与占位符的含义和效果（条目里的小问号「?」会跳转到对应讲解）。
3. **思维链预览** — 按 AI 实际读取顺序自上而下展示；绝对(@深度)条目会可视化地插进「聊天记录」内部。

## 保存

- **应用到当前会话**：立即写入正在使用的预设并刷新原生提示词管理器。
- **保存到预设**：再尝试把整份预设写入当前预设文件。若你的 ST 版本接口不同导致失败，改用「应用」后再点 ST 原生的「更新当前预设」即可。
- **导出 / 导入 JSON**：备份与迁移（导入支持本扩展格式或完整 ST 预设文件）。

## v1.4.2 更新

- **修复手机端弹窗塌成一条黑框**：移动端高度改用视口单位（`100dvh`，回退 `100vh`），不再依赖父层高度解析，弹窗正常全屏铺满。

## v1.4.1 更新

- **手机端适配**：弹窗在移动端改为铺满全屏（用动态视口单位 `dvh`，修复 iOS 下 `100vh` 比可视区高、头尾被裁的问题），头部标签自动换到第二行，底部按钮加大触控区，中间列表可顺畅滚动。电脑端保持居中面板，断点 768px 自动切换。

## v1.4 更新

- **按钮重做**：底部/工具栏按钮改为横排紧凑的设计款，次要操作（导出/导入/放弃）收为图标，标签精简（应用 / 保存 / 添加 / 新建）。修复了中文被挤成竖排一长条的问题。
- **保存更清晰、更可靠**：「保存」= 写入当前预设文件并生效；「应用」= 仅当前会话临时生效（试效果用）。保存在个别 ST 版本下会自动兜底触发原生保存按钮。措辞去掉了费解的「落盘」。
- **防误触**：点击界面外的遮罩、按 Esc 不再关闭窗口，避免编辑内容丢失；仅右上角 ✕ 关闭。
- **点整条展开**：现在点条目任意空白处即可展开编辑（点开关/按钮/拖动节点不受影响）。

## v1.3 更新

- **高级三主题 + 一键切换**：界面重做为自成一体的精致深色面板。右上角三个圆形色板可即时切换主题，选择会被记住：
  - **黑曜 Obsidian**（石墨 + 香槟金，冷峻精密）
  - **墨卷 Manuscript**（深墨 + 古铜，条目名用衬线，编/创作气质）
  - **月白 Platinum**（蓝黑 + 冷铂金，干净中性）
- 统一的设计语言：英文等宽眉标、发微光的脊线 + 环形编号节点、等宽数字、去饱和角色色、精致占位符药丸。

## v1.2 更新

- **统计明细**：点统计条右侧「明细」可展开**按条目 token 占用排行**（从高到低，含占比与可视化条形），一眼看清是哪条吃掉了上下文预算。
- **实时真实 token**：编辑内容时，输入框下方先显示即时估算、停顿后自动替换为 SillyTavern **真实分词**的精确值；各条目行尾标签也会回填真实 token。

## v1.1 更新

- **全新视觉**：左侧脊线 + 编号节点把「AI 读取顺序」变得可见；靛蓝→紫品牌主色、语义化角色色、数据等宽字体。
- **修复可读性**：占位符标签改为实心藏红花药丸 + 深色字，深/浅主题下都清晰（不再是看不清的金字）。
- **字符 / token 统计**：顶部预算条显示启用条目数、总字符、总 token，并按角色分布做可视化；每条目也显示自己的字数/token。token 优先接 SillyTavern **真实分词器**（标「实时分词」），不可用时回退「估算」。

## 兼容性说明

扩展通过 `SillyTavern.getContext()` + 动态 import 多路获取数据，对版本变动有一定容错。若你的 ST 版本里 `oai_settings` / `promptManager` / 预设管理器接口有改动，打开浏览器控制台（F12）能看到带 `[preset-editor]` 前缀的提示，便于定位。
